## Overview

**IDOL-F** (Integrated Detection of Offensive Language Framework) is a multi-stage, graph-structured pipeline for offensive language detection in social media text. It addresses three core limitations of existing systems:

1. **Obfuscation** — users deliberately distort offensive words (e.g., `h@te`, `k!ll`, `d3stroy`) to evade detection
2. **False positives** — models misclassify aggressive-sounding but non-offensive sentences (e.g., *"I will kill this exam"*)
3. **Overconfident predictions** — models assign high-confidence labels to genuinely ambiguous sentences without any uncertainty measure

IDOL-F resolves all three through a 7-phase pipeline that combines syntactic graph reasoning, contrastive intent learning, and conformal uncertainty estimation — the **first application of conformal prediction to offensive language detection**.

---

## Framework Architecture

```
Phase 1: Corpus Construction & Augmentation
         MCA → ACE → IPHNS

Phase 2: Adversarial Text Purification & Semantic Verification
         Lexical PreNorm Surface → ObfDetX → ObfReX → LFA + SDD

Phase 3: Structured Annotated Graph Propagation
         SGRP + OLR → Offensive Predicate Check

Phase 4: Semantic Intent Contrastive Learning
         STC → CICA (InfoNCE + vMF)

Phase 5: Semantic Graph Construction, Propagation & Polarity Separation
         RASGC → ARGP → ICPS

Phase 6: Contextual Offensive Intent Resolution
         DGT → SER → HTV

Phase 7: Conformal Uncertainty Boundary Detection
         CUBD → Classify / Human Analysis / Extended Analysis
```

## Datasets

IDOL-F is trained and evaluated on **5 publicly available datasets**:

| Short Name | Full Name | Size | Source | Labels |
|---|---|---|---|---|
| HASOC | Hate Speech and Offensive Content Identification | ~17,000 | Twitter + Facebook | Binary |
| HSLL | Hate Speech and Offensive Language | ~24,800 | Twitter | 3-class → Binary |
| HTEval | HatEval | ~13,000 | Twitter | Binary |
| HTXplain | HateExplain | ~20,100 | Twitter + Gab | Multi-annotator |
| OLID | Offensive Language Identification Dataset | ~13,000 | Twitter | Binary |

**Cross-Transfer Evaluation Datasets:**

| Name | Description |
|---|---|
| AggPars | Aggression detection dataset |
| TwtPars | Twitter-specific parsing dataset |

> Download each dataset from its official source (HuggingFace / Kaggle) and place the CSV files in `datasets/` following the column naming in `Step_00_Config.py`.

## Configuration

Before running, edit `Step_00_Config.py`:

```python
# Set your local base directory
BASE_DIR = r"C:\path\to\your\IDOL-F"

# Set your OpenAI API key (required for Step 03 — IPHNS)
OPENAI_API_KEY = "sk-..."

# Ablation toggles (set False to disable a component)
ABLATION = {
    "ACE"       : True,
    "IPHNS"     : True,
    "ODMS"      : True,
    "SAGP"      : True,
    "SICL"      : True,
    "ICPS"      : True,
    "COIR"      : True,
    "CUBD"      : True,
```

## Ablation Study

To reproduce the ablation study results, set individual components to `False` in `Step_00_Config.py` and re-run the pipeline. Step 12 automatically collects and reports the performance drop for each disabled component.

Example — ablation without ODMS:

```python
ABLATION = {
    "ACE"   : True,
    "IPHNS" : True,
    "ODMS"  : False,   # ← disable this component
    "SAGP"  : True,
    "SICL"  : True,
    "ICPS"  : True,
```

## Key Novelty Points

| # | Contribution | Details |
|---|---|---|
| 1 | **ODMS** | First obfuscation-aware normalization pipeline with dual verification (LFA + SDD) |
| 2 | **IPHNS** | GPT-4o-Mini synthesized hard negatives to prevent false positives on non-human targets |
| 3 | **SAGP** | Node-level (not sentence-level) offensive predicate detection via dependency graph |
| 4 | **SICL** | Combined InfoNCE + vMF contrastive learning for intent-aware embeddings |
| 5 | **ARGP** | Edge-type-aware GAT with learnable negation attenuation parameter |
| 6 | **ICPS** | Soft GMM clustering for probabilistic intent separation (offensive / non-offensive / uncertain) |
| 7 | **CUBD** | First application of conformal prediction in offensive language detection — statistical coverage guarantee P(y ∈ C(x)) ≥ 1 − α |
